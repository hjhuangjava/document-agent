## 1. 总体架构设计

方案生成智能体：数据/流程驱动，支持历史风格模仿与跨文档关系发现。前后端分离，宏观工作流控制（LangGraph）+ 微观任务自治与技能调用（LangGraph 子图 + ToolNode）结合。

### 1.1 与 v1 的核心变更

| 变更项 | v1 | v2 | 原因 |
|--------|----|----|------|
| Deep Agents 内核 | `deepagents` 包 | LangGraph 子图 + ToolNode | `deepagents` 包不存在，用官方 API 替代 |
| VFS | 无实现 | 基于 `tempfile` + 会话隔离的文件管理 | 必须有明确存储方案 |
| Checkpoint | 未设计 | LangGraph `SqliteSaver` / `PostgresSaver` | 长文档生成需断点恢复 |
| 条件路由 | 硬编码中文匹配 | 结构化校验结果枚举 | 中文匹配脆弱不可靠 |
| 并发模型 | 未设计 | 会话级隔离 + 异步执行 | 多用户场景必需 |
| Node Schema | 缺字段 | 补全 id/version/category/conditions | 前后端交互需要完整协议 |

## 2. 前端系统设计

技术栈：**Next.js (React) + Tailwind CSS + Shadcn UI**

### 2.1 技术选型

| 模块 | 选型 | 理由 |
|------|------|------|
| 应用框架 | Next.js (React) | Vercel AI SDK 生态，流式组件集成 |
| 流程编排画布 | React Flow | 低代码节点连线，直接映射 LangGraph 图结构 |
| 流式交互 | Vercel AI SDK | 原生 SSE 支持，中间推理步骤渲染 |
| 文档编辑器 | TipTap | 可扩展富文本，Markdown 支持，比 Monaco 更适合文档场景 |

### 2.2 用户方案生成端

1. **需求向导舱**：多步骤表单引导用户输入场景参数，支持"一键克隆"历史方案作为写作风格模板。
2. **生成动态看板**：SSE 订阅后端事件流。左侧时间轴展示 Agent 执行进度；右侧打字机式方案预览。
3. **方案工作台**：方案生成后转 Markdown 可编辑状态，AI 辅助润色，Diff 视图对比原始版本与修改版本。
4. **人机反馈组件**：用户微调后点击"反哺库"，修改片段作为 Few-shot 数据异步写入后端知识库。

### 2.3 开发者管理后台

1. **技能仓**：可视化管理工具/技能。
   - 配置界面：在线填写/修改技能元数据（名称、描述、入参 JSON Schema）。
   - 热加载开关：启用/禁用技能，触发后端重新注册 Tool。
2. **流程编排画布**：基于 React Flow 的低代码画布。
   - 拖拽节点、连线、配置参数。
   - 支持**条件分支连线**和**循环回边**（带 max_retries 上限）。
   - 画布导出为标准拓扑 JSON，下发后端。

### 3.1 全局状态设计 (AgentState)

```python
from typing import Annotated, Literal
from typing_extensions import TypedDict
from langgraph.graph.message import add_messages

class ConsistencyResult(TypedDict):
    status: Literal["pass", "fail"]
    violations: list[str]       # 不一致项描述
    retry_count: int            # 当前重试次数

class AgentState(TypedDict):
    messages: Annotated[list, add_messages]     # 对话与推理日志
    business_context: dict                      # 前端输入的业务参数
    virtual_files: dict[str, str]               # 文件名 -> VFS 路径
    consistency_report: ConsistencyResult        # 一致性校验报告（结构化）
    generated_charts: list[dict]                 # 图表数据清单
    current_stage: str                           # 当前阶段标识（供前端进度展示）
```

**v1 变更**：
- `consistency_report` 从 `Dict[str, Any]` 改为 `ConsistencyResult`，条件路由基于 `status` 枚举值而非中文匹配。
- 新增 `current_stage` 字段，前端 SSE 进度展示用。
- 新增 `retry_count` 嵌入 `ConsistencyResult`，循环回边有上限。

### 3.2 虚拟文件系统 (VFS) 设计

```python
import os
import uuid
import shutil
from pathlib import Path

class VirtualFileSystem:
    """会话级隔离的临时文件管理器"""

    def __init__(self, base_dir: str = "/workspace/vfs"):
        self.session_id = str(uuid.uuid4())[:8]
        self.root = Path(base_dir) / self.session_id
        self.root.mkdir(parents=True, exist_ok=True)

    def write(self, filename: str, content: str) -> str:
        filepath = self.root / filename
        filepath.parent.mkdir(parents=True, exist_ok=True)
        filepath.write_text(content, encoding="utf-8")
        return str(filepath)

    def read(self, filename: str) -> str:
        return (self.root / filename).read_text(encoding="utf-8")

    def list_files(self) -> list[str]:
        return [str(p.relative_to(self.root)) for p in self.root.rglob("*") if p.is_file()]

    def cleanup(self):
        """会话结束后清理"""
        shutil.rmtree(self.root, ignore_errors=True)
```

关键设计点：
- **会话隔离**：每个生成任务创建独立 `session_id` 子目录，避免并发覆盖。
- **生命周期**：会话结束（成功或失败）后调用 `cleanup()` 清理，也可配置 TTL 延迟清理。
- **图表文件名**：使用 `uuid` 避免冲突（见 3.3 工具设计）。

#### ④ 图表生成工具 (`tools/chart.py`)

```python
import uuid
import matplotlib
matplotlib.use("Agg")  # 服务器环境必须设置非交互后端
import matplotlib.pyplot as plt
from langchain_core.tools import tool

@tool
def generate_chart(vfs_session_id: str, chart_type: str, data_points: list[dict]) -> str:
    """根据传入数据绘制图表。chart_type 可选 'bar' 或 'pie'。
    返回图表在 VFS 中的路径。"""
    vfs = _get_vfs(vfs_session_id)

    plt.figure(figsize=(6, 4))
    x = [d["label"] for d in data_points]
    y = [d["value"] for d in data_points]

    if chart_type == "bar":
        plt.bar(x, y, color="skyblue")
    elif chart_type == "pie":
        plt.pie(y, labels=x, autopct="%1.1f%%")
    else:
        return f"不支持的图表类型: {chart_type}"

    # 使用 uuid 避免并发文件名冲突
    file_id = uuid.uuid4().hex[:8]
    filename = f"charts/chart_{file_id}.png"
    filepath = vfs.root / filename
    filepath.parent.mkdir(parents=True, exist_ok=True)
    plt.savefig(filepath, format="png", dpi=150)
    plt.close()

    return str(filepath)
```

- 所有工具通过 `vfs_session_id` 访问 VFS，实现会话隔离。

### 3.4 工作流编排实现

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.sqlite import SqliteSaver
from langchain_openai import ChatOpenAI
from langgraph.prebuilt import ToolNode

# --- 模型与工具 ---
model = ChatOpenAI(model="gpt-4o", streaming=True)
tools = [write_section, read_section, query_business_data, check_consistency, generate_chart]
model_with_tools = model.bind_tools(tools)

# --- VFS 注册表 (会话级) ---
_vfs_registry: dict[str, VirtualFileSystem] = {}

def _get_vfs(session_id: str) -> VirtualFileSystem:
    if session_id not in _vfs_registry:
        _vfs_registry[session_id] = VirtualFileSystem()
    return _vfs_registry[session_id]

# --- 图节点 ---
MAX_RETRIES = 3

def research_stage(state: AgentState):
    """数据检索阶段"""
    ctx = state["business_context"]
    result = model_with_tools.invoke([
        ("system", "你是方案编写助手。请调用 query_business_data 查询场景数据。"),
        ("user", f"查询场景 {ctx['scene_id']} 的设备特征值，min_score={ctx.get('min_score', 60)}")
    ])
    return {
        "messages": [result],
        "current_stage": "data_research"
    }

def generation_stage(state: AgentState):
    """章节生成阶段"""
    ctx = state["business_context"]
    result = model_with_tools.invoke([
        ("system", "你是方案编写专家。请参照历史方案风格生成章节，调用 write_section 分段写入。"),
        ("user", f"为场景 {ctx['scene_id']} 生成方案章节，确保字数1500以上。")
    ])
    return {
        "messages": [result],
        "current_stage": "plan_generation"
    }

def validation_stage(state: AgentState):
    """一致性校验阶段"""
    ctx = state["business_context"]
    result = model_with_tools.invoke([
        ("system", "你是校验专家。请调用 check_consistency 核对文档参数。"),
        ("user", f"校验场景 {ctx['scene_id']} 的方案一致性。")
    ])
    return {
        "messages": [result],
        "current_stage": "consistency_validation"
    }

# --- 条件路由 ---
def route_after_validation(state: AgentState) -> str:
    """校验后路由：通过→结束，失败→重试（上限3次）"""
    report = state.get("consistency_report", {})
    retry_count = report.get("retry_count", 0)

    if report.get("status") == "pass":
        return END
    if retry_count >= MAX_RETRIES:
        return END  # 超过重试上限，强制结束
    return "PlanGeneration"  # 退回重写

# --- 构建图 ---
workflow = StateGraph(AgentState)

workflow.add_node("DataResearch", research_stage)
workflow.add_node("PlanGeneration", generation_stage)
workflow.add_node("CriticValidation", validation_stage)
workflow.add_node("Tools", ToolNode(tools))  # 官方 ToolNode 处理工具调用

workflow.add_edge(START, "DataResearch")
workflow.add_edge("DataResearch", "PlanGeneration")
workflow.add_edge("PlanGeneration", "CriticValidation")
workflow.add_conditional_edges("CriticValidation", route_after_validation)

# 带 Checkpoint 的编译
memory = SqliteSaver.from_conn_string(":memory:")
app = workflow.compile(checkpointer=memory)
```

**v1 变更**：
- 函数名与 `add_node` 参数一致。
- 用 `ToolNode` 处理工具调用，而非虚构的 `create_deep_agent`。
- 条件路由基于结构化 `ConsistencyResult.status`，不匹配中文。
- `MAX_RETRIES = 3` 防止无限循环。
- Checkpoint 支持，长任务可断点恢复。

### 3.5 并发与会话管理

```python
import asyncio
from contextlib import asynccontextmanager

# 会话注册表
active_sessions: dict[str, dict] = {}

@asynccontextmanager
async def session_context(session_id: str):
    """会话生命周期管理"""
    vfs = VirtualFileSystem()
    active_sessions[session_id] = {
        "vfs": vfs,
        "status": "running",
        "created_at": asyncio.get_event_loop().time()
    }
    try:
        yield vfs
    finally:
        vfs.cleanup()
        active_sessions.pop(session_id, None)
```

每个用户请求分配独立 `session_id`：
- 独立 VFS 目录（文件隔离）
- 独立 LangGraph Checkpoint thread_id（状态隔离）
- 会话结束自动清理

### 4.4 通用图解释器

```python
import asyncio
from langgraph.graph import StateGraph, START, END

async def universal_node_executor(node_id: str, current_state: AgentState) -> dict:
    """通用 LangGraph 节点：解析并执行前端配置的组件"""

    # 1. 获取节点配置
    node_config = await get_node_config(node_id)
    if not node_config:
        raise ValueError(f"节点配置未找到: {node_id}")

    # 2. 解析输入（state 绑定 / 静态值）
    resolved_inputs = {}
    for inp in node_config["inputs"]:
        bind = inp.get("bind", {})
        if bind.get("type") == "state":
            value = _resolve_state_path(current_state, bind["state_key"])
            if value is None and inp.get("required"):
                raise ValueError(f"必填输入 {inp['name']} 绑定的状态字段 {bind['state_key']} 为空")
            resolved_inputs[inp["name"]] = value
        else:
            resolved_inputs[inp["name"]] = bind.get("value", inp.get("default"))

    # 3. 输入类型校验
    _validate_inputs(resolved_inputs, node_config)

    # 4. 调度执行
    component_type = node_config["component_type"]
    tool = _tool_registry.get(component_type)
    if not tool:
        raise ValueError(f"未注册的工具类型: {component_type}")

    execution_result = await tool.ainvoke(resolved_inputs)

    # 5. 输出映射到 AgentState
    state_updates = {}
    for out in node_config.get("outputs", []):
        state_key = out.get("bind_state_key")
        if state_key:
            _set_state_path(state_updates, state_key, execution_result.get(out["name"]))

    return state_updates


def _resolve_state_path(state: dict, path: str):
    """按点分路径从 state 中取值，如 'business_context.scene_id'"""
    keys = path.split(".")
    current = state
    for key in keys:
        if isinstance(current, dict) and key in current:
            current = current[key]
        else:
            return None
    return current


def _set_state_path(updates: dict, path: str, value):
    """按点分路径写入 updates 字典"""
    keys = path.split(".")
    current = updates
    for key in keys[:-1]:
        current = current.setdefault(key, {})
    current[keys[-1]] = value


def build_graph_from_topology(topology: dict) -> StateGraph:
    """从前端拓扑 JSON 构建 LangGraph StateGraph"""

    workflow = StateGraph(AgentState)

    # 注册节点
    for node in topology["nodes"]:
        nid = node["id"]
        max_retries = node.get("max_retries", 3)

        async def node_fn(state, _nid=nid):
            return await universal_node_executor(_nid, state)

        # 注入 retry 逻辑
        node_fn._max_retries = max_retries
        workflow.add_node(nid, node_fn)

    # 注册边
    for edge in topology["edges"]:
        src = edge["source"]
        tgt = edge["target"]
        condition = edge.get("condition")

        if src == START_NODE:
            workflow.add_edge(START, tgt)
        elif condition:
            # 条件边：需要按条件分组
            pass  # 条件边在下方统一处理
        else:
            workflow.add_edge(src, tgt)

    # 处理条件边（按 source 分组）
    conditional_groups: dict[str, list] = {}
    for edge in topology["edges"]:
        cond = edge.get("condition")
        if cond:
            src = edge["source"]
            conditional_groups.setdefault(src, []).append(edge)

    for src, edges in conditional_groups.items():
        def make_router(edges_list):
            def router(state: AgentState) -> str:
                for e in edges_list:
                    cond = e["condition"]
                    actual = _resolve_state_path(state, cond["field"])
                    if _eval_condition(actual, cond["operator"], cond["value"]):
                        return e["target"]
                return END  # 无条件匹配则结束
            return router

        workflow.add_conditional_edges(src, make_router(edges))

    return workflow


def _eval_condition(actual, operator: str, expected) -> bool:
    ops = {
        "eq": lambda a, b: a == b,
        "ne": lambda a, b: a != b,
        "gt": lambda a, b: a > b,
        "lt": lambda a, b: a < b,
        "gte": lambda a, b: a >= b,
        "lte": lambda a, b: a <= b,
    }
    return ops.get(operator, lambda a, b: False)(actual, expected)
```

**v1 变更**：
- 异步执行 (`ainvoke`)，FastAPI 异步框架必需。
- 输入校验 (`_validate_inputs`)。
- 条件边基于结构化 `condition` 对象，可扩展运算符。
- `build_graph_from_topology` 完整实现拓扑→图的转换。
- 点分路径解析 state，支持嵌套字段。

### 5.1 SSE 流式事件协议

基于 LangGraph `astream_events` 的真实事件格式：

```python
async def stream_generation(session_id: str, business_context: dict):
    config = {"configurable": {"thread_id": session_id}}
    initial_state = {
        "business_context": business_context,
        "current_stage": "init",
        "virtual_files": {},
        "consistency_report": {"status": "pass", "violations": [], "retry_count": 0},
        "generated_charts": [],
    }

    async for event in app.astream_events(
        initial_state,
        config=config,
        version="v2"
    ):
        kind = event["event"]

        if kind == "on_chain_start":
            yield {
                "event": "stage_change",
                "data": {
                    "stage": event["data"].get("input", {}).get("current_stage", "unknown"),
                    "node": event["name"]
                }
            }

        elif kind == "on_tool_start":
            yield {
                "event": "tool_start",
                "data": {
                    "tool": event["name"],
                    "input": event["data"].get("input", {})
                }
            }

        elif kind == "on_tool_end":
            yield {
                "event": "tool_end",
                "data": {
                    "tool": event["name"],
                    "output": str(event["data"].get("output", ""))[:500]  # 截断防溢出
                }
            }

        elif kind == "on_chat_model_stream":
            chunk = event["data"]["chunk"]
            if hasattr(chunk, "content") and chunk.content:
                yield {
                    "event": "text_delta",
                    "data": {"content": chunk.content}
                }
```

**v1 变更**：
- 事件类型对齐 LangGraph `astream_events` v2 真实格式：`on_chain_start`/`on_tool_start`/`on_tool_end`/`on_chat_model_stream`。
- 不再使用虚构的 `on_skill_start`/`on_message_delta`。
